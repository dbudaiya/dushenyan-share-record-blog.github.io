Astro

Astro是什么？

  • 以内容为中心 ：Astro 专为内容丰富的网站而设计。
  • 服务器优先 ：网站在服务器上渲染 HTML 时运行速度更快。
  • 默认快速 ：在 Astro 中构建缓慢的网站是不可能的。
  • 易于使用 ：您不需要成为专家即可使用 Astro 构建某些内容且文档可读性高
  • 功能齐全且灵活 ：超100多种 Astro 集成可供选择。

多页面框架

  • HTML/CSS 优先快速渲染占据
  • 自创 astro 单文件，支持多种文件页面文件
  • 支持各种前端组件（使用时要注意，可能与想象的有所区别，最后有 node ssr 渲染经验）

适合场景（主静态/少交互）

  • 博客
  • 文档
  • 其他的静态页面
  • 其他交互不强的追求速度的页面

前端用 astro 静态渲染

后台管理用 使用 React/Vue/…

后端使用 （express/nestjs/koa…）

开发前奏

  • VScode安装Astro推出的插件Astro
  • 了解部分JavaScript开放API以及TypeScript类型检测
  • 了解vue基础知识包含组件常用响应式API(因为后续会将进行Vue插件引入)
  • Nodejs14.x版本以上

样式

样式可以说是前端写内容页面用的最多的东西

在component目录下,新建Title.astro文件

在主文件index.astro引入


上面能看出在两个组件Title和Layout中都定义h1标签,但两者的样式没有受到影响.
这是因为每个组件style标签是默认含有作用域

想要开启当前组件全局作用域应该在style标签上设置is:global,
这样index.astro的样式就会一起影响到Title.astro中的样式了
